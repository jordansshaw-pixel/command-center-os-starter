# Governance

Status: active governance index
Edition: Community / distributable starter

## Purpose

`_governance/` holds root governance law and the clearly marked evidence that created it.

Short rule:

```text
Load active governance law for operations. Load archives and references only for provenance, audit, or source review.
```

## Active Files

- `steward.md` — canonical root authority for oath, truth, proof, refusal, and correction.
- `atx-build-roadmap.md` — compact active roadmap and how to extend the OS.

## Archives & References

This edition ships without the original build archives and planning references. As you operate, you
may add:

- `source-archive/` for completed audits and exact source records that should not load by default.
- `references/` for planning, research, comparative maps, and design rationale that influenced the OS
  but is not active law.

## No-Load Defaults

Do not load governance archives or references during ordinary work.

Load them only when a route asks for provenance, root design history, or a governance source review.

## Forklift Role

A clean install needs this folder for active governance law. It does not need original planning or
completed audit files to boot.
