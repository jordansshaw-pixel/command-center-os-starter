# Command Center OS — Root Context

Status: compact root context
Edition: Community / distributable starter

## Current State

Command Center OS is a fresh install of an operator's root agentic operating system.

Operating rule:

```text
Startup uses compact runtime kernels and route cards before loading broad source manuals or logs.
```

Runtime Tier 0:

1. `CLAUDE.md`
2. `CONTEXT.md`
3. `_memory/SESSION-BOOT-STATE.md`
4. `_routing/runtime/ROUTING-KERNEL.md`
5. `_routing/runtime/ROUTE-INDEX.md`
6. `_memory/runtime/MEMORY-KERNEL.md`
7. `_memory/runtime/LOAD-INDEX.md`

## First-Run Focus

On a fresh install, complete operator onboarding before opening execution lanes:

1. Run `_routing/runtime/routes/operator-onboarding.md` to populate `_operator/` and `SESSION-BOOT-STATE.md`.
2. Prove an end-to-end route on the sample project (`_examples/`) before adding live projects.
3. Add your own project lanes only after each project can carry its own local truth sources.

## Active Root Pieces

Root operating roles and specialist roles are indexed in:

- `_agents/ROLE-STATUS.json`
- `_agents/ROLE-INDEX.md`
- `_agents/ROLE-INVOCATION-MATRIX.md`

## Project Lanes

No live project lanes ship with this edition. A neutral worked example lives in `_examples/`.
Add project folders at the repo root and keep their operational truth local to the project.

## Recovery

After runtime Tier 0:

1. Select the smallest matching route card.
2. Classify the work.
3. Load only task-specific Tier 1 sources.
4. Load Tier 2 evidence only for a named decision, log, audit trail, role contract, project state, or proof path.
