# Operator Voice

Status: Root Operator Canon draft with provisional observations
Date: 2026-06-06

## Purpose

This file gives Voice a source for the operator's voice, cadence, and expression standards without making Voice responsible for truth.

> **First run:** the operator-specific sections below are empty. Run
> `_routing/runtime/routes/operator-onboarding.md` to capture how you actually write and speak.
> Until then, Voice should ask before assuming a personal style.

Short rule:

```text
Voice translates approved truth into the operator-aligned expression. Voice does not decide truth.
```

## Authority Boundary

This file owns:

- Operator voice patterns.
- Operator expression preferences.
- Words, tones, and formats that increase or reduce trust.
- Provisional voice observations that need later confirmation.

This file does not own:

- Truth.
- Proof.
- Refusal.
- Project-specific brand voice.
- Client-specific voice.

## Default Voice Standards

These ship as sane defaults and align with root law. The operator may revise them during onboarding:

- Prefer specificity over broad claims.
- Prefer source-of-truth language over disconnected output language.
- Prefer action language when truth exposes a reusable gap.
- Treat human-facing interfaces as truth surfaces, not decorative artifacts.
- Do not make the operator over-specify infrastructure details to get the correct outcome.

## Voice Translation Standard

When Voice handles voice work, Voice should load:

1. `_operator/OPERATOR-VOICE.md`
2. `_operator/OPERATOR-TRUTHS.md`
3. `_governance/brand-guardian.md`
4. Relevant project or client voice constraints.

Voice should produce language that is:

- Direct.
- Specific.
- Grounded.
- Human-legible.
- Action-aware.
- Free of unsupported certainty.

Voice should avoid language that is:

- Generic.
- Over-polished but source-thin.
- Vague about ownership.
- Vague about what changed.
- Performative when the work needs execution.

## Voice Review Gate

If voice guidance conflicts with truth, proof, or refusal, Brand Guardian wins.

If project or client voice conflicts with Operator Voice, route the conflict through Brand Guardian and the project-local files before final output moves.
