# Route Index

Status: active Tier 0 runtime
Date: 2026-06-07

## Purpose

Map request signals to compact route cards.

## Routes

| Signal | Route Card |
|---|---|
| First run, empty operator layer, fresh-install boot state | `routes/operator-onboarding.md` |
| New or repaired project/workspace scaffold | `routes/project-scaffold.md` |
| Website, CRM, API, DNS, deploy, repo push, email, calendar, automation, credential | `routes/live-system.md` |
| Decision, memory write, source-improvement, durable finding | `routes/memory-write.md` |
| Truth, proof, brand, public/client wording, unsupported claim | `routes/proof-claims.md` |
| Build, prototype, script, generator, implementation | `routes/build-prototype.md` |
| Resume, pause, transfer, project next owner | `routes/handoff.md` |
| Internal reversible low-risk edit | `routes/fast-lane.md` |
| Multi-boundary or multi-role work | `routes/role-engagement.md` |
| Context loss, startup recovery, broad boot pressure | `routes/context-recovery.md` |
| File placement, stale path, reference classification | `routes/reference-placement.md` |

## Selection Rule

Choose the highest-risk matching card. If several match, start with `role-engagement.md` and let it name downstream cards.
